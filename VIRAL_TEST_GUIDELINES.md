# VIRAL_TEST_GUIDELINES.md

> 테스트홀릭(test-holic-blog) 전용 제작 가이드

이 문서는 테스트홀릭의 자동/수동 게시물 생성 시 반드시 참고하는 운영 기준이다.
목표는 **무난한 심리테스트**가 아니라 **SNS에서 클릭·공유·재방문이 잘 나는 테스트**를 만드는 것이다.

---

## 1. 핵심 목표

테스트홀릭은 하나의 단일 테스트 엔진만 반복해서 사용하지 않는다.
**같은 제목 재질이 아니라, 실제 체감상 다른 테스트처럼 느껴지는 것**이 중요하다.

내부 분기 구조는 여러 방식을 섞어 사용한다.
- 숨김형 MBTI 4축 분기
- 점수형 4결과 분기
- 밸런스 게임형 누적 분기
- 세계관/역할 배정형 분기
- 취향 선택형 분기

즉, 정확성만 추구하는 테스트가 아니라 **재미 + 자기투영 + 공유성 + 다양성**이 강한 테스트를 만든다.

## 2. 운영 전략

### 기본 운영 원칙
테스트홀릭은 아래 두 축을 함께 운영한다.

1. **일상형(범용형) 테스트**
   - 평상시 자동 게시의 기본값
   - 유행을 타지 않고 오래 먹히는 주제 사용
2. **특집형(트렌드형) 테스트**
   - 약 **2주에 1번** 주기로 운영
   - 최근 뜨는 밈/유행어/관계 키워드를 분석해서 반영

### 추천 운영 비율
- **평상시:** 범용형 테스트 중심
- **격주 1회:** 트렌드 분석 기반 특집 테스트

### 중요한 원칙
- 평소 게시물은 너무 유행어에 의존하지 않는다.
- 트렌드형은 고정 리스트를 미리 박아두지 않는다.
- 특집 제작 시점에 **그때 실제로 살아 있는 키워드**를 다시 확인한다.
- 트렌드를 그대로 베끼지 말고, **테스트 언어로 번역해서 사용**한다.

## 3. 테스트 엔진 운영 원칙

### 매우 중요
**모든 테스트를 숨김형 MBTI 엔진으로 만들지 않는다.**

MBTI형은 짧고 효율적이지만, 계속 반복하면 제목만 다르고 체감은 비슷해진다.
그래서 테스트홀릭은 아래 엔진을 섞어 써야 한다.

### A. 숨김형 MBTI 엔진
- 4문항 / 2지선다 / E-I, S-N, T-F, P-J 분기
- 16유형 내부 분기 가능
- 겉으로는 MBTI를 숨긴다
- **사용 빈도:** 일부 시리즈용으로만 사용

### B. 점수형 4결과 엔진
- 4~8문항
- 선택 누적으로 4개 결과 중 하나 도출
- 가장 범용적으로 쓰기 좋음
- 관계/취향/상황형 테스트에 적합

### C. 밸런스 게임형 엔진
- 4문항 내외
- 이지선다 기반
- 캐릭터나 포지션을 빠르게 뽑기 좋음
- 심리검사 느낌보다 게임 느낌이 강함

### D. 세계관/역할 배정형 엔진
- 학교물 / 연애예능 / 판타지 / 회사 / 좀비 아포칼립스 같은 설정 활용
- 결과는 포지션/역할 중심
- 다른 테스트와 체감 차별화가 큼

### E. 취향 선택형 엔진
- 음식 / 여행 / 패션 / 방 / 야식 / 색감 / 플레이리스트 같은 취향 기반
- 심리테스트 같지 않아서 피로감을 줄이기 좋음

### 운영 규칙
- 같은 엔진을 **연속 2회 이상 반복하지 않는 것**을 권장한다.
- 숨김형 MBTI 엔진은 전체 자동 생성물의 일부로만 사용한다.
- 최근 테스트와 제목이 아니라 **체감 구조**가 겹치면 다른 엔진으로 전환한다.

## 4. 주제 선정 원칙

### 범용형에서 우선 사용할 주제 축
1. **썸 / 연애 / 플러팅**
2. **친구들이 보는 나**
3. **단톡방 / 카톡 / DM / SNS 반응**
4. **첫인상 / 분위기 / 숨겨진 매력**
5. **무의식적으로 드러나는 관계 패턴**
6. **술자리 / 모임 / 인간관계 캐릭터**
7. **취향 기반 캐릭터 분류**
8. **상황 대처형 / 밸런스 게임형 / 캐릭터 배정형**

### 트렌드형에서 사용할 주제 축
- 최근 커뮤니티/SNS에서 반복 노출되는 관계 키워드
- 유행하는 감정 표현/밈/자기소개 언어
- 요즘 사람들 사이에서 많이 회자되는 연애/관계 캐릭터 명명법

### 덜 우선인 주제
- 너무 교과서적인 자기이해형
- 지나치게 치유/상담 톤인 주제
- 가치관 설문처럼 느껴지는 주제
- 결과가 지나치게 무난하고 착한 주제

## 5. 장르 운영 원칙

테스트홀릭은 같은 결의 테스트만 반복하지 않는다.

### 권장 장르
- 관계/연애형
- 카톡/단톡/SNS형
- 캐릭터 배정형
- 상황 대처형
- 밸런스 게임형
- 취향 선택형
- 세계관형
- 친구 비교형

### 장르 운영 규칙
- 같은 장르를 **연속으로 2회 이상 반복하지 않는 것**을 권장한다.
- 관계형 다음에는 캐릭터형, 그다음엔 상황형처럼 순환하는 것이 좋다.
- 트렌드형 특집은 범용형과 결이 너무 겹치지 않게 한다.

## 6. 제목 규칙

### 제목 목표
제목은 **설명문**이 아니라 **궁금증 버튼**이어야 한다.

### 좋은 제목 특징
- 사용자가 자기 얘기처럼 느낄 것
- 관계 맥락이 보일 것
- 결과가 궁금해질 것
- 친구에게 보내기 쉬울 것

### 피해야 할 제목
- 나의 인간관계 성향 테스트
- 나의 커뮤니케이션 유형 알아보기
- 내가 선호하는 관계 거리감은?
- 나의 말투 성향 테스트

---

## 7. 결과명 규칙

### 기본 원칙
결과명은 추상적인 상담 용어보다 **캐릭터처럼 불리는 이름**이어야 한다.

### 좋은 결과명 특징
- 짧다
- 말맛이 있다
- 공유 캡처 시 한 줄로 강하다
- 친구가 보고 반응하기 쉽다
- 가능하면 `~재질`, `~포지션`, `~캐릭터`, `인간 ~` 같은 대중적 아키타입 표현을 활용한다.

### 주의할 점
- 특정 유명 IP 실명은 결과명에 직접 쓰지 않는다.
- 대신 대중적으로 이해되는 **아키타입/재질/포지션** 표현으로 치환한다.

## 8. 문항 작성 규칙

### 가장 중요한 원칙
문항은 **설문 문장**이 아니라 **실제 장면**이어야 한다.

### 공통 원칙
- 사용자가 자기 경험을 바로 떠올릴 수 있어야 한다.
- 질문만 보고도 장면이 그려져야 한다.
- 추상적인 자기서술 문장은 피한다.

### 엔진별 작성 요령
#### 숨김형 MBTI 엔진
- 4문항 / 2지선다
- 각 문항은 E-I, S-N, T-F, P-J 축 하나씩 담당
- MBTI를 직접 묻지 않고 상황형으로 숨긴다

#### 점수형 엔진
- 4~8문항
- 4개 결과 중 하나로 자연스럽게 누적되게 설계
- 같은 의미를 돌려 묻지 않도록 주의

#### 밸런스 게임형 엔진
- 선택의 온도차가 확실해야 함
- 질문 자체가 재밌어야 함

#### 세계관형 엔진
- 세계관 몰입이 핵심
- 결과는 역할/포지션 중심으로 간다

#### 취향형 엔진
- 심리검사보다 취향 놀이처럼 느껴져야 함
- 일상 소재 활용

---

## 9. 결과 본문 규칙

### MBTI 결과 노출 규칙
- 숨김형 MBTI 엔진을 쓰더라도 사용자에게는 MBTI를 **표기하지 않는다**.
- 랜딩 페이지, 결과 페이지, 카드 문구 어디에도 MBTI 코드를 노출하지 않는다.
- MBTI는 오직 내부 분기 로직과 제작 설계용으로만 사용한다.

### 결과 페이지 목표
결과는 단순 설명이 아니라 **자기소개 문장처럼 느껴져야** 한다.

---

## 10. 랜딩 문구 규칙

- 랜딩에서는 결과 타입 이름을 미리 공개하지 않는다.
- 랜딩에서는 "A/B/C/D 중 하나" 같은 표현도 쓰지 않는다.
- 랜딩은 무엇을 알 수 있는지만 말하고, 결과 자체는 숨긴다.
- MBTI는 랜딩에서 직접 드러내지 않는다.
- 결과 페이지에서도 MBTI 코드를 직접 표기하지 않는다.

## 11. 트렌드형 특집 운영 규칙

### 주기
- 약 **2주에 1번** 트렌드형 특집 테스트를 운영한다.

### 절차
1. 생성 직전에 최근 트렌드 키워드를 탐색한다.
2. 정말 지금 살아 있는 키워드인지 확인한다.
3. 테스트로 번역 가능한 키워드만 선택한다.
4. 밈을 그대로 베끼지 말고, 테스트홀릭 문법으로 재해석한다.
5. 너무 수명이 짧거나 이미 식은 키워드는 제외한다.

## 12. 중복 방지 규칙

### 매우 중요
제목이 다르다고 다른 테스트가 아니다.

아래 3가지 중 2개 이상이 최근 테스트와 겹치면 **다른 엔진/장르/주제**로 바꾼다.

1. **핵심 분기 구조가 비슷함**
2. **질문 맥락이 비슷함**
   - 예: 카톡/답장/말투/첫인상/관계 거리감만 계속 반복
3. **결과 감정선이 비슷함**
   - 예: 무심함, 배려, 플러팅, 관찰자 포지션만 반복

### 권장 점검 범위
- 최근 5~7개 테스트와 비교
- 제목이 아니라 **체감상 비슷한지**를 점검

## 13. 새 테스트 생성 시 반드시 지킬 것

1. 이 문서를 먼저 읽는다.
2. 오늘 게시물이 범용형인지, 격주 특집형인지 먼저 판단한다.
3. 최근 5~7개 테스트를 보고 엔진/장르/정서 중복을 먼저 피한다.
4. 제목 후보를 3개 이상 생각한 뒤 가장 클릭 강한 것을 고른다.
5. 어떤 테스트 엔진을 쓸지 먼저 정한다.
6. 그 엔진에 맞는 문항 구조를 설계한다.
7. 랜딩 문구에서는 결과 타입과 MBTI를 숨긴다.
8. 결과 페이지에서도 MBTI 코드나 MBTI 언급을 직접 노출하지 않는다.
9. 완성 후 바이럴 체크리스트를 다시 점검한다.

**다음 테스트가 이전 테스트와 다른 재미를 줘야 한다.**
