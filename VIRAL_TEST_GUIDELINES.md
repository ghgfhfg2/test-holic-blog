# VIRAL_TEST_GUIDELINES.md

> 테스트홀릭(test-holic-blog) 전용 제작 가이드

이 문서는 테스트홀릭의 자동/수동 게시물 생성 시 반드시 참고하는 운영 기준이다.
목표는 **무난한 심리테스트**가 아니라 **SNS에서 클릭·공유·재방문이 잘 나는 테스트**를 만드는 것이다.

---

## 1. 핵심 목표

테스트홀릭의 기본 생성 엔진은 **4문항 / 2지선다 / MBTI 4축(E-I, S-N, T-F, P-J) 기반의 숨김형 분기 구조**를 우선 사용한다.
다만 이 구조는 **내부 분기용**으로만 쓰고, 사용자가 보게 되는 겉표현은 MBTI 검사가 아니라 **상황형 바이럴 테스트**여야 한다.

테스트홀릭의 테스트는 아래 4가지를 동시에 노린다.

1. **클릭 욕구**
   - 제목만 보고도 "이건 해보고 싶다"는 반응이 나와야 한다.
2. **완주율**
   - 문항이 설문지처럼 느껴지지 않고, 빠르게 몰입되어야 한다.
3. **공유 욕구**
   - 결과를 캡처해서 스토리/단톡/DM으로 보내고 싶어야 한다.
4. **연속 소비**
   - 하나 하고 나면 다른 테스트도 눌러보게 만들어야 한다.

즉, 정확성만 추구하는 테스트가 아니라 **재미 + 자기투영 + 공유성**이 강한 테스트를 만든다.

## 2. 운영 전략

### 기본 운영 원칙
테스트홀릭은 아래 두 축을 함께 운영한다.

1. **일상형(범용형) 테스트**
   - 평상시 자동 게시의 기본값
   - 유행을 타지 않고 오래 먹히는 주제 사용
2. **특집형(트렌드형) 테스트**
   - 약 **2주에 1번** 주기로 운영
   - 최근 뜨는 밈/유행어/관계 키워드를 분석해서 반영

### 추천 운영 비율
- **평상시:** 범용형 테스트 중심
- **격주 1회:** 트렌드 분석 기반 특집 테스트

### 중요한 원칙
- 평소 게시물은 너무 유행어에 의존하지 않는다.
- 트렌드형은 고정 리스트를 미리 박아두지 않는다.
- 특집 제작 시점에 **그때 실제로 살아 있는 키워드**를 다시 확인한다.
- 트렌드를 그대로 베끼지 말고, **테스트 언어로 번역해서 사용**한다.

## 3. 주제 선정 원칙

### 범용형에서 우선 사용할 주제 축
1. **썸 / 연애 / 플러팅**
2. **친구들이 보는 나**
3. **단톡방 / 카톡 / DM / SNS 반응**
4. **첫인상 / 분위기 / 숨겨진 매력**
5. **무의식적으로 드러나는 관계 패턴**
6. **술자리 / 모임 / 인간관계 캐릭터**
7. **취향 기반 캐릭터 분류**
8. **상황 대처형 / 밸런스 게임형 / 캐릭터 배정형**

### 트렌드형에서 사용할 주제 축
- 최근 커뮤니티/SNS에서 반복 노출되는 관계 키워드
- 유행하는 감정 표현/밈/자기소개 언어
- 요즘 사람들 사이에서 많이 회자되는 연애/관계 캐릭터 명명법

### 덜 우선인 주제
- 너무 교과서적인 자기이해형
- 지나치게 치유/상담 톤인 주제
- 가치관 설문처럼 느껴지는 주제
- 결과가 지나치게 무난하고 착한 주제

## 4. 장르 운영 원칙

테스트홀릭은 같은 결의 테스트만 반복하지 않는다.

### 권장 장르
- 관계/연애형
- 카톡/단톡/SNS형
- 캐릭터 배정형
- 상황 대처형
- 밸런스 게임형
- 취향 선택형
- 세계관형
- 친구 비교형

### 장르 운영 규칙
- 같은 장르를 **연속으로 2회 이상 반복하지 않는 것**을 권장한다.
- 관계형 다음에는 캐릭터형, 그다음엔 상황형처럼 순환하는 것이 좋다.
- 트렌드형 특집은 범용형과 결이 너무 겹치지 않게 한다.

## 5. 제목 규칙

### 제목 목표
제목은 **설명문**이 아니라 **궁금증 버튼**이어야 한다.

### 좋은 제목 특징
- 사용자가 자기 얘기처럼 느낄 것
- 관계 맥락이 보일 것
- 결과가 궁금해질 것
- 친구에게 보내기 쉬울 것

### 권장 제목 패턴
- 나는 왜 사람들이 은근히 못 잊는 사람일까?
- 내가 무심코 주는 플러팅 시그널은?
- 친구들이 느끼는 내 숨은 거리감은?
- 단톡방에서 내가 맡는 포지션은?
- 좋아하는 사람 앞에서 내가 제일 티 나는 순간은?
- 사람들은 왜 나를 생각보다 오래 기억할까?

### 피해야 할 제목
- 나의 인간관계 성향 테스트
- 나의 커뮤니케이션 유형 알아보기
- 내가 선호하는 관계 거리감은?
- 나의 말투 성향 테스트

이런 제목은 너무 보고서/설문지 같다.

## 6. 결과명 규칙

### 기본 원칙
결과명은 추상적인 상담 용어보다 **캐릭터처럼 불리는 이름**이어야 한다.

### 좋은 결과명 특징
- 짧다
- 말맛이 있다
- 공유 캡처 시 한 줄로 강하다
- 친구가 보고 반응하기 쉽다
- 가능하면 `~재질`, `~포지션`, `~캐릭터`, `인간 ~` 같은 대중적 아키타입 표현을 활용한다.

### 좋은 예시
- 인간 공지사항
- 눈치 만렙 재질
- 조용한 실세 포지션
- 단톡방 텐션 메이커
- 무심 플러터 캐릭터
- 분위기 메이커 재질
- 안 친한데도 기억 남는 재질

### 피해야 할 예시
- 조용한 배려형
- 편한 균형형
- 느린 신뢰형
- 계획적 회복형

### 주의할 점
- 특정 유명 IP 실명은 결과명에 직접 쓰지 않는다.
- 대신 대중적으로 이해되는 **아키타입/재질/포지션** 표현으로 치환한다.

## 7. 문항 작성 규칙

### 가장 중요한 원칙
문항은 **설문 문장**이 아니라 **실제 장면**이어야 한다.

사용자는 자기 성향을 분석하고 싶다기보다,
"아 이럴 때 난 진짜 저러지" 하고 바로 고를 수 있어야 한다.

### 권장 문항 스타일
- 카톡/DM 보낼 때
- 단톡에서 반응할 때
- 스토리 올렸을 때/봤을 때
- 소개팅/첫 만남 직후
- 썸 탈 때 답장 텐션
- 술자리/모임에서의 반응
- 친구들 사이에서의 포지션
- 좋아하는 사람 앞에서 망설이는 순간

### 피해야 할 문항 스타일
- 가까운 관계에서 내가 중요하게 생각하는 것은?
- 내가 선호하는 상호작용 방식은?
- 내 가치관과 가장 가까운 것은?

### 문항 구성 규칙
- 기본 4문항
- 기본값은 **2지선다**로 작성한다.
- 각 문항은 아래 MBTI 축 하나를 담당한다.
  1. E / I
  2. S / N
  3. T / F
  4. P / J
- 문항은 MBTI를 직접 묻지 말고, **상황형 장면**으로 숨겨서 작성한다.
- 최소 1~2문항은 **메신저/SNS/단톡 등 디지털 맥락**을 포함한다.
- 최소 1문항은 사용자가 "들킨 느낌"을 받는 디테일이 있어야 한다.
- 짧더라도 각 문항의 훅이 강해야 하며, 설문지처럼 느껴지는 추상 문항은 피한다.

## 8. 선택지 작성 규칙

### 선택지 목표
선택지는 유형 설명이 아니라 **사람이 실제로 할 법한 반응**처럼 보여야 한다.
기본적으로 한 문항당 **2개 선택지**를 사용하고, 각 선택지는 해당 축의 양끝(E/I, S/N, T/F, P/J)을 담당한다.

### 해야 할 것
- 반응형 문장으로 쓸 것
- 서로 결이 달라 보이게 쓸 것
- 각 선택지가 너무 노골적인 MBTI 설명처럼 보이지 않게 쓸 것
- 같은 축이라도 테스트 컨셉에 맞게 말맛을 바꿀 것

### 하지 말아야 할 것
- 선택지가 MBTI 축 질문처럼 직접적으로 보이는 것
- 모든 문항이 동일한 어조/길이/구조
- 선택지만 읽어도 유형 의도가 바로 보이는 문장
- "나는 신중하다 / 나는 다정하다 / 나는 밝다" 식의 자기서술 남발

## 9. 결과 본문 규칙

### MBTI 결과 노출 규칙
- 내부적으로는 16유형(MBTI 코드)로 분기할 수 있다.
- 하지만 사용자에게는 MBTI를 **표기하지 않는다**.
- 랜딩 페이지, 결과 페이지, 카드 문구 어디에도 MBTI 코드를 노출하지 않는다.
- MBTI는 오직 내부 분기 로직과 제작 설계용으로만 사용한다.

### 결과 페이지 목표
결과는 단순 설명이 아니라 **자기소개 문장처럼 느껴져야** 한다.

### 구성 원칙
각 결과는 아래 흐름을 따른다.
1. 첫 문단: "나 이거다" 싶은 핵심 훅
2. 중간: 왜 이런 분위기가 나오는지 설명
3. 마지막: 매력 포인트 / 사람들이 느끼는 인상 / 공유하고 싶은 한 줄

---

## 10. 랜딩 문구 규칙

- 랜딩에서는 결과 타입 이름을 미리 공개하지 않는다.
- 랜딩에서는 "A/B/C/D 중 하나" 같은 표현도 쓰지 않는다.
- 랜딩은 무엇을 알 수 있는지만 말하고, 결과 자체는 숨긴다.
- MBTI는 랜딩에서 직접 드러내지 않는다.
- 결과 페이지에서도 MBTI 코드를 직접 표기하지 않는다.

## 11. 트렌드형 특집 운영 규칙

### 주기
- 약 **2주에 1번** 트렌드형 특집 테스트를 운영한다.

### 절차
1. 생성 직전에 최근 트렌드 키워드를 탐색한다.
2. 정말 지금 살아 있는 키워드인지 확인한다.
3. 테스트로 번역 가능한 키워드만 선택한다.
4. 밈을 그대로 베끼지 말고, 테스트홀릭 문법으로 재해석한다.
5. 너무 수명이 짧거나 이미 식은 키워드는 제외한다.

### 원칙
- 미리 트렌드 주제 목록을 고정해두지 않는다.
- 실제 제작 시점에 다시 확인한다.
- 트렌드형도 결국은 테스트홀릭 톤에 맞게 다듬는다.

## 12. 바이럴 관점 체크리스트

### 제목

- 설명형 제목이 아닌가?
- 친구에게 보내기 쉬운가?

### 문항
- 4문항 안에 지루함 없이 끝나는가?
- 모든 문항이 실제 장면처럼 그려지는가?
- 사용자가 자기 경험을 즉시 떠올릴 수 있는가?
- 각 문항이 E/I, S/N, T/F, P/J 축 중 하나와 명확히 연결되는가?

### 선택지
- 선택지가 MBTI 축 노출처럼 보이지 않는가?
- 선택지가 유형 설명문처럼 보이지 않는가?
- 2개 보기 모두 실제 반응처럼 자연스러운가?

### 결과
- 결과명이 약하지 않은가?
- 결과 첫 문장이 캡처하고 싶을 정도로 센가?
- 친구가 봤을 때 반응할 포인트가 있는가?
- MBTI를 숨겨도 캐릭터형 결과만으로 충분히 재미가 살아나는가?

### 전체
- 너무 착하고 무난하기만 하지 않은가?
- "헐 이거 너다" 반응이 나올 만한가?
- 최근 게시물들과 장르/결이 너무 겹치지 않는가?

## 13. 새 테스트 생성 시 반드시 지킬 것

1. 이 문서를 먼저 읽는다.
2. 오늘 게시물이 범용형인지, 격주 특집형인지 먼저 판단한다.
3. 제목 후보를 3개 이상 생각한 뒤 가장 클릭 강한 것을 고른다.
4. 내부적으로 어떤 MBTI 16유형 구조를 사용할지 먼저 정한다.
5. 결과 노출용 캐릭터형 이름 또는 포장 방식을 먼저 정한다.
6. 4문항을 만들고, 각 문항이 E/I, S/N, T/F, P/J 중 어떤 축인지 명확히 대응되게 설계한다.
7. 선택지는 2지선다로 작성하되 MBTI 질문처럼 직접 보이지 않게 만든다.
8. 랜딩 문구에서는 결과 타입과 MBTI를 숨긴다.
9. 결과 페이지에서도 MBTI 코드나 MBTI 언급을 직접 노출하지 않는다.
10. 최근 게시물과 장르가 너무 겹치지 않는지 확인한다.
11. 완성 후 바이럴 체크리스트를 다시 점검한다.

## 14. 최종 한 줄 원칙

테스트홀릭의 테스트는
**"정확한 검사"보다 "재밌고 공유하고 싶은 자기 캐릭터 콘텐츠"**에 가깝다.

속은 MBTI 구조를 활용할 수 있어도,
겉은 반드시 **바이럴형 상황 테스트**처럼 보여야 하며,
사용자에게는 MBTI가 직접 보이지 않아야 한다.
